# 交易决策流程

> 触发词：能买吗/能卖吗/要不要/买入/卖出/下单/加仓/减仓

## 步骤 1：确认操作

"你想做什么？买还是卖？哪只股票？多少？"

## 步骤 2：加载框架

Read 以下文件（并行）：
- `05-工具层/sell-discipline.md` — 卖出信号检查
- `05-工具层/behavioral-finance.md` — 当前的偏见是什么？
- `01-决策系统/universal-decision-framework.md` — 通用决策五问
- `06-执行层/pre-trade-checklist.md` — 60 秒自检卡片

## 步骤 3：逐条跑检查

### 买入前检查（9 条）
1. 这个公司做什么的？我能用两句话跟朋友讲清楚吗？
2. 它的护城河是什么？5 年后还在吗？
3. 当前估值的反向 DCF：价格在赌什么？
4. 我比市场多知道什么？（诚实回答）
5. 如果买了跌 20%，我会加仓还是割肉？（如果是割肉 → 别买）
6. 买完后，仓位会超过 20% 吗？（超过 → 必须减仓计划）
7. 今天涨了多少？（> 5% → 不追）
8. 博弈论一问：如果这是错的，谁在另一边？他们为什么卖？
9. 博弈论二问：如果大部分人都觉得这是好主意，凭什么我能赚钱？

### 卖出前检查（5 条）
1. 买入逻辑变了吗？（没变 → 别卖）
2. 到止损线了吗？（-8% → 减半，-15% → 清仓）
3. PE 超过历史 90% 分位了吗？（超过 → 考虑卖）
4. 是因为跌了害怕才想卖？（是 → 加载行为金融，识别偏见）
5. 卖了之后买什么？（没想好 → 别卖）

## 步骤 4：给判断

```
🟢 可以操作 — 所有检查通过
🟡 等等看 — 有 1-2 条不通过，建议观望
🔴 不建议 — 3 条以上不通过，放弃这次操作
```

## 步骤 5：确认执行

"确认要操作吗？我帮你记录到交易日志。"

## 步骤 6：写日志

交易完成后，追加到 `knowledge/08-日志/trade-log.md`：
- 日期 + 买入/卖出 + 价格 + 数量 + 买入逻辑一句话 + 事前验尸
