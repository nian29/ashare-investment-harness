# 日报更新前时间检查

**Why:** 2026-06-18 更新日报时，在 A 股盘中/刚收盘阶段反复搜索收盘数据（十几次 WebSearch），数据尚未更新到搜索索引，浪费时间。以后必须先确认时间。

**How to apply:**

## 更新日报前强制三步

```
第一步：确认今天是交易日吗？
  → 查交易日历或搜索 "A股 2026年X月X日 是否开市"
  → 周末/节假日 → 不需要更新日报，告知用户今日休市

第二步：现在几点？
  → 9:30 前 → 盘前，只有昨日数据
  → 9:30-11:30 → 盘中，只有实时/午盘数据
  → 11:30-13:00 → 午休，上午收盘数据可能已出
  → 13:00-15:00 → 下午盘，实时数据
  → 15:00-17:00 → 刚收盘，数据正在发布和索引中，可能不全
  → 17:00 后 → 收盘数据基本就绪，最佳更新时间

第三步：数据能拿到吗？
  → 盘中 → 用 AKShare 实时接口（get_stock_info_xueqiu等），不用 WebSearch 搜收盘
  → 收盘后 2 小时内 → 可能需要等，告知用户"数据正在更新中"
  → 收盘后 2 小时+ → WebSearch + AKShare 历史数据都可
```

## 用户说"更新日报"时的标准回复

```
场景 A（盘中）："现在是 XX:XX，A 股还在交易中。我先用盘中数据做分析，收盘后可以再补精确数字。"
场景 B（刚收盘）："刚收盘，数据可能还没完全出来。我可以先用板块资金流和盘中走势做分析，精确收盘价稍后核实。"
场景 C（非交易日）："今天休市，不需要更新日报。上次交易日是 X 月 X 日。"
场景 D（收盘后 2h+）：正常执行完整日报流程。
```

## 关键原则

- **不要反复搜不存在的数据**——搜 3 次拿不到就承认数据还没出来，用已有信息分析
- **盘中 ≠ 不能分析**——板块资金流、盘中走势、消息面都有价值，只是不说"收盘价"
- **诚实 > 假装有数据**——标注"估"和"数据说明"是负责任的，比凭空编数字好
